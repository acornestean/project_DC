Both links use the same site permission add-on which was generated via the form from AMO PROD.

The below prefs must be set in about:config. After setting the prefs, a browser restart will be required.

  • dom.webmidi.enabled = true
  • midi.testing = true

The first link proposes to test the distribution control mechanism in which installation will be disallowed for add-ons which, even though have a valid origin, the add-on is hosted somewhere else.

The second link proposes to test the scenario in which the add-on has a valid origin and is hosted on the origin domain. As such, installation will be allowed and the permission will be granted to the domain post installation.

Add-on with valid origin but hosted somewhere else - installation will be DISALLOWED

Add-on with valid origin and hosted on origin domain - installation will be ALLOWED
